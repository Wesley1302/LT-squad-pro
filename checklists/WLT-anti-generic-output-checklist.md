# Anti-Generic Output Checklist (cross-cutting)

Rode em QUALQUER saída antes de entregar ao usuário.

- [ ] Nada de resposta genérica ("depende", "pode ser", "vamos explorar")
- [ ] Usa dados reais do state (avatar, oferta, produto) — não placeholders
- [ ] Nenhuma prova/pesquisa/depoimento inventado
- [ ] Hipótese está marcada como hipótese (não como fato)
- [ ] Diferencia: existe-no-metodo | convencao-observada | extensao-lt-squad-pro
- [ ] Se falta input essencial → PAROU e pediu o dado (não preencheu com genérico)
- [ ] Força especificidade (números, nomes, palavras do avatar)
- [ ] Tom direto, sem floreio, foco em decisão/execução

**Decisão:** PASS → entrega · FAIL → reescrever com especificidade ou pedir o dado ausente
