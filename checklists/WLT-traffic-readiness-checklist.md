# Traffic Readiness Checklist (BLOQUEANTE)

- [ ] Produto validado no state
- [ ] Página validada (page.validation_status=pass)
- [ ] Mínimo 5 criativos prontos (ideal 10+)
- [ ] WLT-tracking-readiness-checklist em PASS (Pixel + CAPI + evento de compra + UTMs + validação)
- [ ] Estrutura: 1 campanha CBO, 1 conjunto Advantage+ (abertão, só Brasil, sem segmentar idade/gênero), 5-20 criativos
- [ ] Orçamento = nº criativos × (ticket/2)
- [ ] Metas definidas ANTES: ROAS 2.0+, CPV ≤R$2,70, CPC ≤R$23, CAC < ticket
- [ ] Regras de decisão 1-3-7 documentadas (dia 1 só observa)
- [ ] Teste separado de escala

**Decisão:** PASS → subir teste / WLT-funnel-diagnostician · FAIL → completar pré-requisito · BYPASS → "quero pular com risco documentado"
