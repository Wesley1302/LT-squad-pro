# Landing Page Implementation Checklist

Estratégia
- [ ] Produto, avatar, promessa, oferta, big idea, mecanismo, prova, objeções presentes (do state)
- [ ] 12 dobras na ordem
- [ ] CTA verbal, garantia, FAQ

UI/UX
- [ ] Premium (refined gradients, subtle glassmorphism, clean hierarchy)
- [ ] Mobile-first, desktop responsive, conversion-focused
- [ ] Mockups, cards de entregáveis, blocos de prova, seção de ancoragem, CTA sticky opcional

Animações
- [ ] GSAP (não GASP), ScrollTrigger, Three.js quando fizer sentido
- [ ] Mobile-safe; nada de animação pesada em mobile; preserva velocidade

Engenharia
- [ ] HTML/CSS/JS ou framework escolhido, componentes organizados
- [ ] Responsividade, acessibilidade, performance, SEO básico
- [ ] Sem dependências desnecessárias, assets placeholders, deploy, README, validação visual

Restrições
- [ ] Não inventa depoimentos
- [ ] Preço só na dobra 9
- [ ] CTA "Quero acessar agora" (não "Comprar")
- [ ] Sem texto genérico (usa language_bank)

**Decisão:** PASS → entrega o prompt de landing · FAIL → corrigir prompt · BYPASS → não se aplica
