# Offer Obviousness Checklist (BLOQUEANTE)

- [ ] Promessa específica (resultado + prazo)
- [ ] Prazo claro
- [ ] Objeção principal removida
- [ ] Big idea diferenciada (não clichê)
- [ ] Mecanismo crível
- [ ] Entregáveis concretos (ícone + nome + descrição, quantidades)
- [ ] Order bumps VALIDADOS no WLT-order-bump-checklist (etapa obrigatória pós-produto)
- [ ] Order bump COMPLETA (não compete) — ângulo análise/execução/expansão, ~1/3 do preço
- [ ] Upsell definido (~3x, pós-compra)
- [ ] Prova real OU plano ético de obtenção (NUNCA inventada)
- [ ] CTA verbal ("Quero acessar agora"), nunca "Comprar"
- [ ] Ancoragem lógica (preços riscados + mockup tudão)
- [ ] Filtro de comprador errado ("é pra você que..." / "não é pra você que...")
- [ ] A oferta parece ÓBVIA demais pra recusar

**Decisão:** PASS → WLT-deliverable-builder / WLT-sales-page-builder · FAIL → refazer o bloco · BYPASS → frase exata
