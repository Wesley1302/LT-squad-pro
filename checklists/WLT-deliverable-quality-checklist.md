# Deliverable Quality Checklist

- [ ] Cabe no miojo (consumível/aplicável em <48h)
- [ ] Sem curva de aprendizagem
- [ ] Entrega o quick win prometido
- [ ] Formato escolhido é o de menor atrito para o resultado
- [ ] Escopo tem critério de "pronto" claro
- [ ] Aponta qual task de prompt de produção usar
- [ ] Micro app / agente: caso de uso e inputs/outputs claros

**Decisão:** PASS → WLT-sales-page-builder / WLT-ai-production-prompt-engineer · FAIL → recortar escopo · BYPASS → frase exata
