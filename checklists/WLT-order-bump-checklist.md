# Order Bump Checklist (BLOQUEANTE)

Rodar em CADA bump:

- [ ] Completa o produto principal (próximo passo do MESMO problema)
- [ ] NÃO compete com o produto principal
- [ ] Barato o suficiente pra marcar sem pensar (~1/3 do principal; contexto > regra cega)
- [ ] Percepção de valor imediata
- [ ] Relação direta com a intenção de compra
- [ ] Entrega simples
- [ ] Aumenta ticket médio SEM confundir a promessa
- [ ] Não transforma a oferta em bagunça
- [ ] Explicável em uma frase
- [ ] Nome autoexplicativo
- [ ] Ângulo identificado (análise | execução | expansão)

**Decisão:** PASS → WLT-build-offer-promise (oferta completa) · FAIL → WLT-build-order-bumps (refazer/cortar bump) · BYPASS → "quero pular com risco documentado"
